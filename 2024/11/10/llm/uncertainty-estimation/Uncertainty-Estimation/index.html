[ACL2024 Long Paper] Shifting Attention to Relevance: Towards the Predictive Uncertainty Quantification of Free-Form Large Language Models

Motivation

与 prompt 相关的 token 在不确定性估计中是否比不相关的 token 更为关键?

论文以 Predictive Entropy 为 Baseline, 通过引入 Relevance 和 Uncertainty Proportion 来回答上面的问题. 前者用来衡量 token 或句子语义相关度, 越高代表语义越重要; 后者用来衡量 token 或句子的不确定性, 越高代表承载的不确定性价值越高

Predictive Entropy 的计算公式为

PE(s,x)=−log⁡p(s∣x)=−∑ilog⁡p(zi∣s<i,x)PE(\boldsymbol{s}, \boldsymbol{x}) = -\log p(\boldsymbol{s}|\boldsymbol{x}) = -\sum_{i}\log p(z_i|\boldsymbol{s}_{<i}, \boldsymbol{x}) PE(s,x)=−logp(s∣x)=−i∑​logp(zi​∣s<i​,x)

在 token 级别, 有

RT(zi,s,x)=1−∣g(x∪s,x∪s\{zi})∣UPT(zi,s,x)=−log⁡p(zi∣s<i,x)PE(s,x)\begin{aligned} R_T(z_i, \boldsymbol{s}, \boldsymbol{x}) &= 1-|g(\boldsymbol{x}\cup\boldsymbol{s}, \boldsymbol{x}\cup\boldsymbol{s}\backslash \{z_i \})|\\ UP_T(z_i, \boldsymbol{s}, \boldsymbol{x}) &= \frac{-\log p(z_i|\boldsymbol{s}_{<i}, \boldsymbol{x})}{PE(\boldsymbol{s}, \boldsymbol{x})} \end{aligned} RT​(zi​,s,x)UPT​(zi​,s,x)​=1−∣g(x∪s,x∪s\{zi​})∣=PE(s,x)−logp(zi​∣s<i​,x)​​

公式里的 “\” 代表集合的差集

这里 x\mathbf{x}x 代表输入的 prompt, ziz_izi​ 代表第 iii 个 token, s\boldsymbol{s}s 代表句子, x\mathbf{x}x 代表输入, g(⋅,⋅)g(\cdot, \cdot)g(⋅,⋅) 计算两个句子的相似度, 具体而言论文使用 Cross-Encoder-RoBERTa-large 方法

在 sentence 级别, 有

RS(s,S,x)=∑j=1,j≠ig(si,j)p(sj∣x)UPS(s,S,x)=PE(si,x)∑kPE(sk,x)\begin{aligned} R_S(\boldsymbol{s}, S, \boldsymbol{x}) &= \sum_{j=1, j\neq i} g(\boldsymbol{s}_i, \boldsymbol{j})p(\boldsymbol{s}_j|\boldsymbol{x})\\ UP_S(\boldsymbol{s}, S, \boldsymbol{x}) &= \frac{PE(\boldsymbol{s}_i, \boldsymbol{x})}{\sum_k PE(\boldsymbol{s}_k, \boldsymbol{x})} \end{aligned} RS​(s,S,x)UPS​(s,S,x)​=j=1,j=i∑​g(si​,j)p(sj​∣x)=∑k​PE(sk​,x)PE(si​,x)​​

这里 SSS 代表包括所有句子的集合, s\boldsymbol{s}s 代表句子, p(sj∣x)p(\boldsymbol{s}_j|\boldsymbol{x})p(sj​∣x) 代表句子 sj\boldsymbol{s}_jsj​ 的概率, 1≤k≤K1\leq k\leq K1≤k≤K, 1≤i,j≤K1\leq i,j\leq K1≤i,j≤K

直观上看, token-level 的 Relevance 反映了 token ziz_izi​ 在句子 s\boldsymbol{s}s 中的语义重要性, 计算了去除该 token 前后句子语义的变化幅度, 再用 1 减去上值; Uncertainty Proportion 反映了 token ziz_izi​ 承载的不确定性价值, 计算了 token ziz_izi​ 的朴素熵占整个句子不确定性的比例. sentence-level 同理

依次计算每个 token 或句子的 Relevance 和 Uncertainty Proportion, 得到如下结果: 仅有少量 token 具有较大的 Relevance 和 Uncertainty Proportion; Relevance 越小, Uncertainty Proportion 越大, 说明承载语义信息少的 token 承载的不确定性价值更高 (某种二次证明, 符合直觉)

Conclusion: 与 prompt 相关的 token 在不确定性估计中比不相关的 token 更为关键, 应当给予更高的权值

Method

token-level

Shifting Attention to Relevance (SAR). 定义句子 sj={z1,z2,…,zNj}\boldsymbol{s}_j = \{z_1, z_2, \dots, z_{N_j} \}sj​={z1​,z2​,…,zNj​​}, NjN_jNj​ 表示句子 sj\boldsymbol{s}_jsj​ 的长度. 基于 Motivation 部分的公式, 将句子内部不同 token 的 Relevance 进行归一化. 包含 NjN_jNj​ 个 token 的句子 sj\boldsymbol{s}_jsj​ 的 Relevance 为

R~T(zi,sj,x)=RT(zi,sj,x)∑nNjRT(zn,sj,x)\tilde{R}_T(z_i, \boldsymbol{s}_j, \boldsymbol{x}) = \frac{R_T(z_i, \boldsymbol{s}_j, \boldsymbol{x})}{\sum_{n}^{N_j} R_T(z_n, \boldsymbol{s}_j, \boldsymbol{x})} R~T​(zi​,sj​,x)=∑nNj​​RT​(zn​,sj​,x)RT​(zi​,sj​,x)​

然后向语义重要的 token 增加权值, 修改句子的 Predictive Entropy 计算公式为

ET(zi,sj,x)=−log⁡p(zi∣s<i,x)R~T(zi,sj,x)E_T(z_i, \boldsymbol{s}_j, \boldsymbol{x}) = -\log p(z_i|\boldsymbol{s}_{<i}, \boldsymbol{x})\tilde{R}_T(z_i, \boldsymbol{s}_j, \boldsymbol{x}) ET​(zi​,sj​,x)=−logp(zi​∣s<i​,x)R~T​(zi​,sj​,x)

于是得到整个段落进行 token 语义偏移后的熵为

sentence-level

语义上一致的句子比其他句子更具说服力, 所以可以通过提高语义一致的句子的生成概率来减少句子的不确定性

这里 p(sj∣x)=∏ip(zi∣<i,xp(\boldsymbol{s}_j|\boldsymbol{x}) = \prod_i p(z_i|\boldsymbol{<i}, \boldsymbol{x}p(sj​∣x)=∏i​p(zi​∣<i,x 是句子 sj\boldsymbol{s}_jsj​ 的生成概率, ttt 控制偏移幅度的温度系数 (代码里默认是 0.001). 于是得到整个段落进行 sentence 语义偏移后的熵为 (KKK 为段落中的句子数)

sentence-level 进行偏移的方法比较粗糙. 计算单个句子的熵时, 作者在句子出现的概率上加了一个有关于 Relevance 的项, 通过这一项的温度系数来控制偏移幅度
为什么这个温度系数可以控制偏移幅度呢? 这里公式是熵的形式, 真数 (也就是对数函数的自变量处) 进行线性增长, 线性增长的系数控制函数的增长速度. 在该语境下, (对单一句子而言的) 语义一致的句子的 Relevance 本身更大, 从而乘系数后得到的增幅更大, 进而与语义不一致的句子的熵的差值更大 (感觉说的有点糊涂了, 下面的例子更直接吧)
类比, 2*10=20 和 4*10=40, 虽然乘数相同, 但是结果差距变得更大了, 从而增加了真数部分的差值, 进而提高了语义一致句子的熵与语义不一致句子的熵的差值

SENTSAR(S,x)=1K∑kES(sk,S,x)\mathrm{SENT}\mathit{SAR}(S, \boldsymbol{x}) = \frac{1}{K}\sum_k E_S(\boldsymbol{s}_k, S, \boldsymbol{x}) SENTSAR(S,x)=K1​k∑​ES​(sk​,S,x)

值得注意的是, sentence-level 计算单一句子的熵形式与语义熵相似, 本文作者提出语义熵 (ICLR2023) 在平均超过 20 个 token 的长句中表现不理想, 而本方法会更好. orz, 平均超过 20 个token, 这个长度有点长了?

Overall Measurement

token-level 和 sentence-level 的两种偏移处理方法各有不同, 但他们是正交的, 可以自然地结合起来. 为此, 只需将 sentence-level 方法的每个句子的生成概率替换成 token-level 方法的偏移概率即可. 也即

[ICLR2025 Workshop] How Many Opinions Does Your LLM Have? Improving Uncertainty Estimation in NLG

语义熵使用多项式采样, 多项式采样会倾向于生成多个序列的相似样本, 这在计算上是低效的, 且一些语义类可能被忽略. 基于此, 论文提出 SDLG (Semantic Diverse Language Generation), 搜索那些具有高概率且具有高语义多样性的输出序列. 论文代码未开源

具体而言, 文章提出计算 Attribution score, Substitution score, Importance score. Attribution score 根据推理的梯度, 确定哪个输出序列中的 token 应该被更改; 该分数表示当 token yiy_iyi​ 被更改时, 对语义的影响越大. Substitution score 表示用词表的哪个 token 替换更好; 该分数越高表示 Embedding 的变化与语义变化更一致. Importance score 表示在给定上下文情况下, 用替代 token 替换当前 token 的可能性; 该分数越高表示替换当前 token 时的可能性越大/重要性越高.

计算完所有替代 token viv_ivi​ 的三个分数后, 根据分数对潜在替代项排序, 然后用排名最高的替代 token 替换既有生成 token, 然后按原有解码方式 (一般式多项式采样) 继续生成过程.

MLE Estimator for LID

Layer Selection and Distance-aware MLE

在上一节, 讨论了如何计算 Xi\mathbf{X}^iXi 的 LIDs. 在大模型中计算的话, 存在两个问题

  1. 每一层的表示维度为 DDD, 所以很难说用哪个是最优的
  2. MLE 假定密度函数 fff 是恒定的, 这在 Causal LLMs 不成立

解决方法有两种

Layer Selection

LLMs 在每层为每个 token 生成一个 DDD 维的表示. 这里选择 Xi\mathbf{X}^iXi 的最后一个 token 即 X−1i\mathbf{X}^i_{-1}X−1i​ 作为这一层的表示

但是实验证明, 最后一层 layer 的表示可能不是最具信息量的特征 (LIDs 预测真实性的性能与测试集上 LIDs 绝对值的总和有很好的相关性, 但在某些层上有所偏移). 这里建议选择 lll 层

[ICML2024 Oral] Decomposing Uncertainty for Large Language Models through Input Clarification Ensembling

一句话概括: 本文聚焦于偶然性不确定性, 提出了一种和 Deep Emsemble 对称的不确定性度量方式, 通过对输入进行"澄清", 从而减少输入的偶然不确定性. Whitebox 和 Blackbox 均可以使用

Uncertainty Decomposition 是将预测模型的不确定性分解为偶然不确定性(数据不确定性)和认知不确定性(模型不确定性), 前者是数据生成汇总固有的随机性引起的, 后者是模型训练数据缺失引起的. 既有的 UD 研究主要研究偶然不确定性

X\boldsymbol{X}X 和 Y\boldsymbol{Y}Y 代表任务的输出输出, θ\thetaθ 代表 LLM 的参数; p(Y∣X)p(\boldsymbol{Y}|\boldsymbol{X})p(Y∣X) 和 q(Y∣X,θq(\boldsymbol{Y}|\boldsymbol{X}, \boldsymbol{\theta}q(Y∣X,θ 分别代表真实分布和模型分布. 总不确定性 total uncertainty 是预测分布的熵 U=H(q(Y∣X;θ))\mathcal{U} = \mathcal{H}(q(\boldsymbol{Y}|\boldsymbol{X; \boldsymbol{\theta}}))U=H(q(Y∣X;θ)), 由偶然不确定性 aleatoric uncertainty UD\mathcal{U}_DUD​ 和 认知不确定性 epistemic uncertainty UE\mathcal{U}_EUE​ 组成

贝叶斯神经网络 Bayesian Neural Network(2015) 和深度集成 Deep Ensembles(2017)

贝叶斯神经网络 (BNN) 对神经网络的参数分布进行建模, 给定训练数据, 可以通过变分推断近似后验分布. 这种方法计算成本很高;

深度集成有更好的扩展性, 维护 K 个模型, 每个模型的参数为 θ(k)\boldsymbol{\theta}^{(k)}θ(k), 最小化每个模型训练的损失函数, 相当于完成优化问题 min⁡KL(p(Y∣X)∥q(Y∣X,θ)).\min \mathrm{KL}(p(\boldsymbol{Y}|\boldsymbol{X})\|q(\boldsymbol{Y}|\boldsymbol{X}, \boldsymbol{\theta})).minKL(p(Y∣X)∥q(Y∣X,θ))., 不同模型的初始值不同, 优化后的参数也不同, 所有模型的参数可看成采样结果, 根据采样结果估计参数的分布 p(θ∣D)p(\boldsymbol{\theta}|\mathcal{D})p(θ∣D), 这里 D\mathcal{D}D 是训练数据集. 所以贝叶斯网络的集成分布可以表示为 q(Y∣X)=Eq(θ∣D)[q(Y∣X,θ)].q(\boldsymbol{Y}|\boldsymbol{X}) = \mathbb{E}_{q(\boldsymbol{\theta}|\mathcal{D})} [q(\boldsymbol{Y}|\boldsymbol{X}, \boldsymbol{\theta})].q(Y∣X)=Eq(θ∣D)​[q(Y∣X,θ)]., 所以不确定量可以表示为

\mathcal{H}(q(\boldsymbol{Y}|\boldsymbol{X}) = \underbrace{\mathcal{I}(\boldsymbol{Y}; \boldsymbol{\theta}|\boldsymbol{X})}_{\textcircled1} + \underbrace{\mathbb{E}_{q(\boldsymbol{\theta}|\mathcal{D})} [\mathcal{H}(q(\boldsymbol{Y}|\boldsymbol{X}, \boldsymbol{\theta}))]}_{\textcircled2}

这里 L\mathcal{L}L 是在 qqq 分布下的互信息. 第一项用于衡量不同模型见的分歧, 第二项用来衡量单独模型的平均不确定性, 两项可以近似于认知不确定性和偶然不确定性

上述两种方法常用来衡量模型的不确定性, 但是对 LLMs 是否依然适合呢? 作者做了实验, 发现深度集成并没有显著减小认知不确定性, 所以这两种方法失效了. 进一步的, 作者提出了一种对称于 BNN 方法的替代框架

Input Clarification Ensembling

既然集成模型以减小认知不确定性比较困难, 那能修改输入来减小认知不确定性吗?

Excuse? 这样减小的不是偶然不确定性吗

  1. Input Clarification: 给定输入 X\boldsymbol{X}X, 生成一系列 “澄清”(clarification) C(k)\boldsymbol{C}^{(k)}C(k), 这些澄清用于减小输入带来的认知不确定性. 加入澄清后的输入表示为 X⊕Ck\boldsymbol{X}\oplus \boldsymbol{C}^kX⊕Ck, 这里 ⊕\oplus⊕ 代表 concat 操作. 需要说明的是, {C(k)}\{\boldsymbol{C}^{(k)}\}{C(k)} 是一个集合
  2. Ensemble: 输入澄清的分布表示为 q(C∣X)q(\boldsymbol{C}|\boldsymbol{X})q(C∣X), 模型输出变化为 q(Y∣X)=Eq(C∣X)[q(Y∣X⊕C,θ)]q(\boldsymbol{Y}|\boldsymbol{X}) = \mathbb{E}_{q(\boldsymbol{C}|\boldsymbol{X})}[q(\boldsymbol{Y}|\boldsymbol{X}\oplus\boldsymbol{C}, \boldsymbol{\theta})]q(Y∣X)=Eq(C∣X)​[q(Y∣X⊕C,θ)]

    \mathcal{H}(q(\boldsymbol{Y}|\boldsymbol{X})) = \underbrace{\mathcal{I}(\boldsymbol{Y}; \boldsymbol{C}|\boldsymbol{X})}_{\textcircled1'} + \underbrace{\mathbb{E}_{q(\boldsymbol{\boldsymbol{C}|\boldsymbol{X}})}\mathcal{H}(q(\boldsymbol{Y}|\boldsymbol{X}\oplus \boldsymbol{C}))}_{\textcircled2'}

    上面 \textcircled1' 代表模型输出分布与澄清间的互信息, 可以近似输入模糊性引起的偶然不确定性; \textcircled2' 代表给定不同澄清时的模型输出分布的平均熵, 表示给定澄清后的不确定性, 也即认知不确定性 (这里假设输入模糊性是认知不确定性的唯一来源)

值得注意的是, 以上公式与 Deep Ensemble 的公式具有对称性, 二者分别关注偶然不确定性和认知不确定性, 这分别对应各自公式的第二项
