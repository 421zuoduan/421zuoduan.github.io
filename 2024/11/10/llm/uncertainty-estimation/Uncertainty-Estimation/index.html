Shifting Attention to Relevance: Towards the Predictive Uncertainty Quantification of Free-Form Large Language Models

Motivation

与 prompt 相关的 token 在不确定性估计中是否比不相关的 token 更为关键?

论文以 Predictive Entropy 为 Baseline, 通过引入 Relevance 和 Uncertainty Proportion 来回答上面的问题. 前者用来衡量 token 或句子语义相关度, 越高代表语义越重要; 后者用来衡量 token 或句子的不确定性, 越高代表承载的不确定性价值越高

Predictive Entropy 的计算公式为

PE(s,x)=−log⁡p(s∣x)=−∑ilog⁡p(zi∣s<i,x)PE(\boldsymbol{s}, \boldsymbol{x}) = -\log p(\boldsymbol{s}|\boldsymbol{x}) = -\sum_{i}\log p(z_i|\boldsymbol{s}_{<i}, \boldsymbol{x}) PE(s,x)=−logp(s∣x)=−i∑​logp(zi​∣s<i​,x)

在 token 级别, 有

RT(zi,s,x)=1−∣g(x∪s,x∪s\{zi})∣UPT(zi,s,x)=−log⁡p(zi∣s<i,x)PE(s,x)\begin{aligned} R_T(z_i, \boldsymbol{s}, \boldsymbol{x}) &= 1-|g(\boldsymbol{x}\cup\boldsymbol{s}, \boldsymbol{x}\cup\boldsymbol{s}\backslash \{z_i \})|\\ UP_T(z_i, \boldsymbol{s}, \boldsymbol{x}) &= \frac{-\log p(z_i|\boldsymbol{s}_{<i}, \boldsymbol{x})}{PE(\boldsymbol{s}, \boldsymbol{x})} \end{aligned} RT​(zi​,s,x)UPT​(zi​,s,x)​=1−∣g(x∪s,x∪s\{zi​})∣=PE(s,x)−logp(zi​∣s<i​,x)​​

公式里的 “\” 代表集合的差集

这里 x\mathbf{x}x 代表输入的 prompt, ziz_izi​ 代表第 iii 个 token, s\boldsymbol{s}s 代表句子, x\mathbf{x}x 代表输入, g(⋅,⋅)g(\cdot, \cdot)g(⋅,⋅) 计算两个句子的相似度, 具体而言论文使用 Cross-Encoder-RoBERTa-large 方法

在 sentence 级别, 有

RS(s,S,x)=∑j=1,j≠ig(si,j)p(sj∣x)UPS(s,S,x)=PE(si,x)∑kPE(sk,x)\begin{aligned} R_S(\boldsymbol{s}, S, \boldsymbol{x}) &= \sum_{j=1, j\neq i} g(\boldsymbol{s}_i, \boldsymbol{j})p(\boldsymbol{s}_j|\boldsymbol{x})\\ UP_S(\boldsymbol{s}, S, \boldsymbol{x}) &= \frac{PE(\boldsymbol{s}_i, \boldsymbol{x})}{\sum_k PE(\boldsymbol{s}_k, \boldsymbol{x})} \end{aligned} RS​(s,S,x)UPS​(s,S,x)​=j=1,j​=i∑​g(si​,j)p(sj​∣x)=∑k​PE(sk​,x)PE(si​,x)​​

这里 SSS 代表包括所有句子的集合, s\boldsymbol{s}s 代表句子, p(sj∣x)p(\boldsymbol{s}_j|\boldsymbol{x})p(sj​∣x) 代表句子 sj\boldsymbol{s}_jsj​ 的概率, 1≤k≤K1\leq k\leq K1≤k≤K, 1≤i,j≤K1\leq i,j\leq K1≤i,j≤K

直观上看, token-level 的 Relevance 反映了 token ziz_izi​ 在句子 s\boldsymbol{s}s 中的语义重要性, 计算了去除该 token 前后句子语义的变化幅度, 再用 1 减去上值; Uncertainty Proportion 反映了 token ziz_izi​ 承载的不确定性价值, 计算了 token ziz_izi​ 的朴素熵占整个句子不确定性的比例. sentence-level 同理

依次计算每个 token 或句子的 Relevance 和 Uncertainty Proportion, 得到如下结果: 仅有少量 token 具有较大的 Relevance 和 Uncertainty Proportion; Relevance 越小, Uncertainty Proportion 越大, 说明承载语义信息少的 token 承载的不确定性价值更高 (某种二次证明, 符合直觉)

Conclusion: 与 prompt 相关的 token 在不确定性估计中比不相关的 token 更为关键, 应当给予更高的权值

Method

token-level

Shifting Attention to Relevance (SAR). 定义句子 sj={z1,z2,…,zNj}\boldsymbol{s}_j = \{z_1, z_2, \dots, z_{N_j} \}sj​={z1​,z2​,…,zNj​​}, NjN_jNj​ 表示句子 sj\boldsymbol{s}_jsj​ 的长度. 基于 Motivation 部分的公式, 将句子内部不同 token 的 Relevance 进行归一化. 包含 NjN_jNj​ 个 token 的句子 sj\boldsymbol{s}_jsj​ 的 Relevance 为

R~T(zi,sj,x)=RT(zi,sj,x)∑nNjRT(zn,sj,x)\tilde{R}_T(z_i, \boldsymbol{s}_j, \boldsymbol{x}) = \frac{R_T(z_i, \boldsymbol{s}_j, \boldsymbol{x})}{\sum_{n}^{N_j} R_T(z_n, \boldsymbol{s}_j, \boldsymbol{x})} R~T​(zi​,sj​,x)=∑nNj​​RT​(zn​,sj​,x)RT​(zi​,sj​,x)​

然后向语义重要的 token 增加权值, 修改句子的 Predictive Entropy 计算公式为

ET(zi,sj,x)=−log⁡p(zi∣s<i,x)R~T(zi,sj,x)E_T(z_i, \boldsymbol{s}_j, \boldsymbol{x}) = -\log p(z_i|\boldsymbol{s}_{<i}, \boldsymbol{x})\tilde{R}_T(z_i, \boldsymbol{s}_j, \boldsymbol{x}) ET​(zi​,sj​,x)=−logp(zi​∣s<i​,x)R~T​(zi​,sj​,x)

于是得到整个段落进行 token 语义偏移后的熵为

TOKENSAR(sj,x)=∑iNjET(zi,sj,x)\mathrm{TOKEN}\mathit{SAR}(\boldsymbol{s}_j, \boldsymbol{x}) = \sum_{i}^{N_j} E_T(z_i, \boldsymbol{s}_j, \boldsymbol{x}) TOKENSAR(sj​,x)=i∑Nj​​ET​(zi​,sj​,x)

sentence-level

语义上一致的句子比其他句子更具说服力, 所以可以通过提高语义一致的句子的生成概率来减少句子的不确定性

ES(sj,S,x)=−log⁡(p(sj∣x)+1tRS(sj,S,x))=−log⁡(p(sj∣x)+∑k≠jg(sj,sk)p(sk∣x)t⏟sentence relevance)\begin{aligned} E_S(\boldsymbol{s}_j, S, \boldsymbol{x}) &= -\log \left( p(\boldsymbol{s}_j|\boldsymbol{x}) + \frac{1}{t}R_S(\boldsymbol{s}_j, S, \boldsymbol{x})\right)\\ &= -\log \left(p(\boldsymbol{s}_j|\boldsymbol{x}) + \underbrace{\frac{\sum_{k\neq j}g(\boldsymbol{s}_j, \boldsymbol{s}_k)p(\boldsymbol{s}_k|\boldsymbol{x})}{t}}_{\text{sentence relevance}}\right) \end{aligned} ES​(sj​,S,x)​=−log(p(sj​∣x)+t1​RS​(sj​,S,x))=−log⎝⎜⎜⎛​p(sj​∣x)+sentence relevancet∑k​=j​g(sj​,sk​)p(sk​∣x)​​​⎠⎟⎟⎞​​

这里 p(sj∣x)=∏ip(zi∣<i,xp(\boldsymbol{s}_j|\boldsymbol{x}) = \prod_i p(z_i|\boldsymbol{<i}, \boldsymbol{x}p(sj​∣x)=∏i​p(zi​∣<i,x 是句子 sj\boldsymbol{s}_jsj​ 的生成概率, ttt 控制偏移幅度的温度系数 (代码里默认是 0.001). 于是得到整个段落进行 sentence 语义偏移后的熵为 (KKK 为段落中的句子数)

sentence-level 进行偏移的方法比较粗糙. 计算单个句子的熵时, 作者在句子出现的概率上加了一个有关于 Relevance 的项, 通过这一项的温度系数来控制偏移幅度
为什么这个温度系数可以控制偏移幅度呢? 这里公式是熵的形式, 真数 (也就是对数函数的自变量处) 进行线性增长, 线性增长的系数控制函数的增长速度. 在该语境下, (对单一句子而言的) 语义一致的句子的 Relevance 本身更大, 从而乘系数后得到的增幅更大, 进而与语义不一致的句子的熵的差值更大 (感觉说的有点糊涂了, 下面的例子更直接吧)
类比, 2*10=20 和 4*10=40, 虽然乘数相同, 但是结果差距变得更大了, 从而增加了真数部分的差值, 进而提高了语义一致句子的熵与语义不一致句子的熵的差值

SENTSAR(S,x)=1K∑kES(sk,S,x)\mathrm{SENT}\mathit{SAR}(S, \boldsymbol{x}) = \frac{1}{K}\sum_k E_S(\boldsymbol{s}_k, S, \boldsymbol{x}) SENTSAR(S,x)=K1​k∑​ES​(sk​,S,x)

值得注意的是, sentence-level 计算单一句子的熵形式与语义熵相似, 本文作者提出语义熵 (ICLR2023) 在平均超过 20 个 token 的长句中表现不理想, 而本方法会更好. orz, 平均超过 20 个token, 这个长度有点长了?

语义熵的计算公式如下

SE(x)=−∑cp(c∣x)log⁡p(c∣x)=−∑c((∑s∈cp(s∣x))log⁡[∑s∈cp(s∣x)])\begin{aligned} SE(x) &= -\sum_c p(c|x)\log p(c|x)\\ &= -\sum_c \left(\left(\sum_{\mathbf{s}\in c} p(\mathbf{s}|x)\right) \log\left[\sum_{\mathbf{s}\in c} p(\mathbf{s}|x)\right] \right) \end{aligned} SE(x)​=−c∑​p(c∣x)logp(c∣x)=−c∑​((s∈c∑​p(s∣x))log[s∈c∑​p(s∣x)])​

主要区别在于, 负对数概率乘的权值不同, 语义熵是直接乘以句子的生成概率, 而本文是乘以句子的 Relevance

Overall Measurement

token-level 和 sentence-level 的两种偏移处理方法各有不同, 但他们是正交的, 可以自然地结合起来. 为此, 只需将 sentence-level 方法的每个句子的生成概率替换成 token-level 方法的偏移概率即可. 也即

p′(si∣x)=e−TOKENSAR(si,x)p′(sj∣x)=e−TOKENSAR(sj,x)ET,S(sj,S,x)=−log⁡(p′(si∣x)+∑k≠jg(sj,k)p′(sj∣x)t\begin{aligned} p'(\boldsymbol{s}_i|\boldsymbol{x}) &= e^{-\mathrm{TOKEN}\mathit{SAR}(\boldsymbol{s}_i, \boldsymbol{x})}\\ p'(\boldsymbol{s}_j|\boldsymbol{x}) &= e^{-\mathrm{TOKEN}\mathit{SAR}(\boldsymbol{s}_j, \boldsymbol{x})}\\ E_{T,S}(\boldsymbol{s}_j, S, \boldsymbol{x}) &= -\log (p'(\boldsymbol{s}_i|\boldsymbol{x}) + \frac{\sum_{k\neq j}g(\boldsymbol{s}_j, \boldsymbol{k})p'(\boldsymbol{s}_j|\boldsymbol{x})}{t} \end{aligned} p′(si​∣x)p′(sj​∣x)ET,S​(sj​,S,x)​=e−TOKENSAR(si​,x)=e−TOKENSAR(sj​,x)=−log(p′(si​∣x)+t∑k​=j​g(sj​,k)p′(sj​∣x)​​

因此, KKK 个句子的 token- and sentence-level shifted predictive entropy 为

SAR=1K∑kET,S(sk,S,x)\mathit{SAR} = \frac{1}{K}\sum_k E_{T, S}(\boldsymbol{s}_k, S, \boldsymbol{x}) SAR=K1​k∑​ET,S​(sk​,S,x)

附录关于温度系数 ttt 的消融实验表明, 使用 t=1×10−3t=1\times10^{-3}t=1×10−3 和 t=1×100t=1\times10^0t=1×100 区别不大. 感觉这个 sentence-level 的偏移方法还是粗糙了一些, 可以优化.
